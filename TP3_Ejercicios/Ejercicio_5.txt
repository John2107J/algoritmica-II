Ejercicio 5
a) Si demuestro que un algoritmo es O(n²) en el peor caso, ¿es posible que sea O(n) para algunos datos?
Sí, es posible.
O(n²) en el peor caso significa que existe una cota superior cuadrática para las entradas más desfavorables. Pero para entradas particulares (por ejemplo, un array ya ordenado en un algoritmo de ordenamiento), el algoritmo puede ejecutarse en O(n) pasos.
Ejemplo: Insertion Sort es O(n²) en el peor caso, pero O(n) cuando la entrada ya está ordenada.

b) Si demuestro que un algoritmo es O(n²) en el peor caso, ¿es posible que sea O(n) para todos los datos?
Sí, también es posible (aunque la cota sería floja).
Si el algoritmo es O(n) para todos los datos, entonces en particular también es O(n²) para todos los datos, ya que O(n) ⊆ O(n²). La cota O(n²) simplemente no sería ajustada.

c) Si demuestro que un algoritmo es Θ(n²) en el peor caso, ¿es posible que sea O(n) para algunos datos?
Sí, es posible.
Θ(n²) en el peor caso dice que las entradas más desfavorables tienen comportamiento exactamente cuadrático. Pero esto no impide que para ciertas entradas específicas (caso favorable), el algoritmo corra en O(n).
El peor caso y el comportamiento en entradas particulares son cosas distintas.

d) Si demuestro que un algoritmo es Θ(n²) en el peor caso, ¿es posible que sea O(n) para todos los datos?
No, es imposible.
Si fuera O(n) para todos los datos, entonces en particular también lo sería para las entradas del peor caso. Pero Θ(n²) en el peor caso implica que esas entradas requieren al menos cn² pasos (por la cota inferior Ω(n²)), lo cual contradice ser O(n).
